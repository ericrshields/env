If new machine, copy PowerShell settings into
    C:\Users\<USER>\AppData\Local\Packages\Microsoft.WindowsTerminal_8wekyb3d8bbwe\LocalState\settings.json

Setup WSL and install Ubuntu
    Guide: https://learn.microsoft.com/en-us/windows/wsl/install
    Open PowerShell
    Enable WSL, if needed: "dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart"
    Install default Ubuntu: "wsl --install -d Ubuntu"
    Restart computer and let the install finish (May take around 10 minutes)
    Check if the distro is installed: "wsl --list"
    If not, re-run the install command
    Create username and password (pick something you can type when needed for sudo)
    "sudo apt update" (Updates package lists - may take a while)
    "sudo apt full-upgrade"
    Useful tools to install: "sudo apt install tree dtrx"
        tree: Modern directory tree viewer (replaces legacy shell script)
        dtrx: Intelligent archive extractor (replaces legacy extract script)
        Note: git bash-completion is included with git package automatically

Install essential CLI tools
    "sudo apt install jq ripgrep fd-find bat"
        jq       - JSON processor for working with JSON files/APIs
        ripgrep  - Fast grep alternative (command: rg) - 10-100x faster than grep
        fd-find  - Fast find alternative (command: fdfind) - simpler syntax, respects .gitignore
        bat      - cat with syntax highlighting (command: batcat)
    "sudo snap install yq"  (YAML processor - similar to jq but for YAML files)
    Create symlinks for Ubuntu-specific naming:
        "mkdir -p ~/.local/bin"
        "ln -sf $(which fdfind) ~/.local/bin/fd"
        "ln -sf $(which batcat) ~/.local/bin/bat"

Set up SSH
    Copy SSH key to ~/.ssh/
    Start ssh process: eval $(ssh-agent)
    Add GitHub key to ring: ssh-add ~/.ssh/<github_key>.osh

Setup env
    (PowerShell recommended over screen for local dev) "sudo apt install screen"
    "git clone git@github.com:ericrshields/env.git ~/env"
    Ensure everything is in local user permission scope: "sudo chown -R eshields:eshields ~"
    "perl ~/env/install.pl"
    If you don't need the backups: "find . -name "*~*""
    Confirm good to delete, then "find . -name "*~*" -delete"
    "source ~/.bashrc"
    Close terminal and reopen
    If things seem broken, ensure that the latest changes to env (and sub repos!) are pushed to origin
    "mkdir ~/projects"
    Clone desired repos into projects dir (don't forget to re-add the ssh key to the ring)
    Sometimes the .ssh files will have permissions altered. Reset with "chmod 600 ~/.ssh/<key file>"

Optional: Migrate to Modern Prompt (Starship)
    Note: The install.pl script sets up gitprompt.pl by default, but modern alternatives are faster.
    See docs/gitprompt-performance-analysis.md for detailed performance comparison.

    Install Starship (recommended for most users):
        "curl -sS https://starship.rs/install.sh | sh"
        Follow the prompts (installs to /usr/local/bin/starship)

    Configure Starship in ~/.bashrc:
        Comment out or remove the gitprompt PROMPT_COMMAND section (lines 24-28 in bash/rc)
        Add at the end of ~/.bashrc: eval "$(starship init bash)"

    Optional: Customize Starship prompt
        Create config: "mkdir -p ~/.config && starship preset nerd-font-symbols > ~/.config/starship.toml"
        Edit ~/.config/starship.toml to customize appearance
        See: https://starship.rs/config/

    Test performance:
        "cd ~/projects/grafana && time git status"
        Should be ~50ms (faster than gitprompt.pl's ~68ms)

    Alternative for Zsh users: Powerlevel10k
        "git clone --depth=1 https://github.com/romkatv/powerlevel10k.git ~/powerlevel10k"
        Add to ~/.zshrc: source ~/powerlevel10k/powerlevel10k.zsh-theme
        Run configuration wizard: "p10k configure"

Install node, nvm, and npm
    Install nvm via curl command: https://github.com/nvm-sh/nvm/blob/master/README.md#installing-and-updating
    Refresh the .rc file "source ~/.bashrc"
    "nvm install node" (This will install the lastest version of both nodejs and npm)
    "nvm use node"
    "nvm alias default node"

// Set up SSH for use with <Ubuntu> Windows Subsystem for Linux (WSL)
// Note: Can also run "sudo su" to avoid individual sudo commands
// Source: https://www.youtube.com/watch?v=7wVX-9XkasM
sudo apt-get install openssh-server
sudo vi /etc/ssh/sshd_config
    // Change "PasswordAuthentication" to "yes"
    // Add "AllowUsers <username>" to the end of the file
sudo service ssh start
sudo visudo
    // Add "%sudo ALL=NOPASSWD: /usr/sbin/sshd" after "%sudo ALL=(ALL:ALL) ALL"
// PuTTY to "localhost" via SSH
